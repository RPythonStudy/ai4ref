# AI4REF 프로젝트 개요

## 목적
생성형 인공지능, R/Python 및 PostgreSQL 등 오픈소스 도구를 활용하여, 의료데이터 분석 기반 의학연구 논문작성 전 과정의 생산성 향상 및 자동화

## 방향성
- 자동화 + 필요시 수작업(Hybrid) 구조로 정확성과 생산성 동시 추구
- 단계별 기능은 개별 실행도, 연계 실행도 가능하도록 설계 (모듈화, 유연한 파이프라인)

## 전체 프로세스 및 단계별 개발 방향

### Collector (수집기)
역할:
- 사용자 검색문구와 AI 확장 검색어로 PubMed 등에서 논문/참고문헌 자동 수집
- Zotero 등 서지관리 시스템에 자동 저장

아웃풋: 원문(PDF/링크), 메타데이터(JSON/YAML), 서지정보

### Preprocessor (전처리/필터링기)
역할:
- 수집된 논문의 중복, 저품질, 불필요 자료 필터링
- 기준(연도, 저널, Open Access 등)별 필터 적용
- 서지정보 정제 및 표준화

아웃풋: 정제된 논문 리스트 및 메타데이터

### Summarizer (요약기)
역할:
- AI 기반 논문 요약(구조적 요약: Abstract, Methods 등)
- PICO 등 주요 연구정보 자동 추출

아웃풋: 논문별 요약본, 표준화된 주요 정보

### Analyzer (분석기)
역할:
- 정보 기반 통계/메타분석
- 질적평가(Risk of Bias 등) 및 데이터 시각화
- 분석 결과 해설문 생성

아웃풋: 분석 보고서, 표, 그래프, 해설문

### Reviewer/Editor (검수/수정기)
역할:
- AI 결과물의 사용자 피드백 및 수작업 검수
- fact-check, 중복/오류/오해 수정
- (필요시) AI ↔ 사용자 상호작용 반복 (human-in-the-loop)

아웃풋: 최종 검수된 결과물

### Exporter (내보내기/논문작성기)
역할:
- 결과물의 다양한 포맷(Word, LaTeX, Markdown, .qmd 등) 자동 변환
- 참고문헌 자동 포맷팅 및 논문 초안 자동 생성

아웃풋: 논문초안, 참고문헌, 분석/부속자료

## 개발/운영 지침
- 각 단계는 src/{collector, preprocessor, summarizer, analyzer, reviewer, exporter} 등 모듈화된 폴더로 분리
- 각 모듈은 독립 실행(CLI, REST API 등) 및 연계 파이프라인 실행 모두 지원
- 입력/출력은 파일 또는 DB를 표준 인터페이스로 사용
- 단계별 파라미터는 config 파일(yaml/json 등) 기반으로 관리
- Docker 등 컨테이너 배포도 지원, 필요시 개별 서비스화 가능
- hybrid(자동+수작업) 검수/수정 구조 적극 지원
- 로그/예외처리 일관성 확보
- 결과물의 reproducibility(재현성)와 추적성(traceability) 확보
- 확장성(인터페이스/API/웹UI 등) 고려
